# <span style="color: yellow ">Инструкция для работы с Git и локальными репозиториями </span>

*GIT — распределённая система управления версиями, созданная Линусом Торвальдсом для управления разработкой ядра Linux и в настоящее время получившая очень широкое распространение в среде разработчиков программного обеспечения.*

----------------------------------
## 1. Проверка наличия установленного Git.
----------------------------------
_Открыть терминал с помощью меню
Viev --> Terminal или сочетания помощью клавиш **Ctrl + `**_ 
* В терминале  выполнить команду:
```
git --version
```
* Если Git установлен, появится сообщение с информацией о версии.
Иначе будет сообщение об <span style="color:red"> ошибке </span>.
-----------------------------------
## 2. Установка Git. 
-----------------------------------
* Загружаем последнюю версию Git с сайта [git-scm.com](https://git-scm.com/downloads).
* Устанавливаем с настройками по умолчанию. 
------------------------------------
## 3. Настройки Git.
------------------------------------
* При первом знакомстве с Git нужно представиться.
Для этого в терминале следует ввести 2 команды:

```
git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```

* Для проверки текущих регистрационных данных (знакомства)
введите команду:


```
git config --global --list
```


--------------------------------
## 4. Инициализация репозитория
--------------------------------
* Для этого открываем локальную папку в меню File --> Open Folder или с помощью сочетания клавиш **Ctrl + O**. После этого:

_Открываем терминал с помощью меню Viev --> Terminal или с помощью клавиш **Ctrl + `**_

Вводим команду:

```
git init
```
-----------------------------------
## 5. Создаём новый файл 
-----------------------------------
*Создаем новый файл при помощи меню File --> New File.*

* Имя файла пишем с расширением  .md.
Например: **`instruction.md`**

------------------------------------
## 6. Добавляем изменения файла в репозиторий.
------------------------------------
_Предварительно сохраняем файл при помощи меню File --> Safe или с помощью сочитанья клавиш **Ctrl + S**._

>Так же можно поставить автосохранение в меню File --> Avto Save

После:

* В терминале вводим команду:

```
git add <имя_файла>
```
* Тем самым вносим изменения в текущий документ.

------------------------------------
## 7. Добавление коммита (версии изменений).
------------------------------------
* Для добавления коммита (версии изменений) в терминале следует ввести следующую команду:
```
git commit -m <Комментарий к коммиту>
```
Где `-m` это message/Сообщение

После этого вы получите подобное сообщение:

![Сообщение](commit.png)

------------------------------------
## 8. Просмотр версий - коммитов (версии изменения) 
------------------------------------
* Для просмотра коммитов (версий изменений) после добавления изменений в репозиторий можно произвести  в терминале с помощью команды:

```
git log
```
>*или если не нужна вся информации об измениях то можно воспользоваться командой:* 

```
git log --oneline
````
коммиты (версии изменений) будут предоставлены в виде одной строки.

> _После просмотра коммитов (версий изменений) в некоторых случаях когда есть строка END в терминале, для продолжения работы требуется нажать клавишу **Q**_

------------------------------------
## 9. Переход в выбранный коммит (версию изменений)

------------------------------------

* Для перехода в выбранный коммит (версию изменений) достаточно копировать 4-6 символов названия нужного коммита (версии изменений) и ввести следующую команду в терминале:

```
git checkout <4-6 символов коммита>
```
* После чего вы попадете на стадию тех изменений которые были сохранены в данном коммите (версии изменений).

*Для возвращения к актуальной версии* можно воспользоваться следующей коммандой:

```
git checkout master
```
-----------------------------------
## 10. Отображение статуса файла
-----------------------------------
* Для того чтобы вывести текущий статут файла нужно воспользоваться командой:

```
git status
```
------------------------------------
## 11. Отслеживание изменений текущего состояния файла и предыдущей фиксаций.
------------------------------------
* Для отслеживания изменений в текущем файле до сохранения и внесения изменений в коммит (версии изменений) воспользуйтесь командой:

```
git_diff
```
где:

**<span style="color: red "> - удалённые строки </span>**

**<span style="color: green "> + добавленные строки </span>**

--------------------------------------
## 12. Игнорирование файлов.

* Для того чтобы исключить из отслеживания репозитория определенные фаилы и папки нужно создать новый фаил с именем файла `.gitignore`

* И в него внести название файлов или шаблонов, соответствующие данным файлам или папкам.

>_Для того что бы игнорировать определенный тип файлов на пример images.jpg или images.png и не прописывать каждое имя. Можно в файл .gitignore прописать расширение файла таким образом:_

```
*.jpg или *.png
```
---------------------------------------
## 13.Создание новой ветки в репозитории.
---------------------------------------

* Для создания новой ветки в репозитории в нужно ввести следующую команду:
```
git branch <имя ветки>
```
* Для проверки и отображения всех созданных и существующих веток используйте команду:
```
git branch
```
>_По умолчанию в Git главная или корневая ветка называется **master**._

* Для быстрого создания новой ветки и перехода на новую ветку можно произвести командой:

```
git checkout -b <имя_новой_ветки>
```
* Так же мы можем посмотреть список всех коммитов с отображением всех веток графически с помощью команды:
```
git log --graph --oneline
``` 
где --graph (отображение графически)
    --oneline (в одну линию без доп.информации)

![отображение веток](graph.png)

--------------------------------------
## 14. Переход на новые ветки, сохранение и внесение измений. Ошибки и способы решений. 
---------------------------------------
С помощью команды:
```
git branch
```
Просматриваем все ветки в нашем репозитории. Ветка в которой мы находимся отмечена (`*`).

Для перехода в нужную ветку введите команду:
```
git checkout <имя_нужной_ветки>
```
_**Примечание:**_
*При переходе на новую ветку могут возникнуть проблемы. В том случае если в текущей ветке не сохранены изменения и не создан новый коммит. Для проверки состояния используйте команду `git status`. Если изменения не сохранены тогда воспользуйтесь командой `git add` и `git commit -m` или `git commit -am`. Как описанно 6 и 7 пункте данной инструкции*

--------------------------------------------
## 15. Слияние (соединение) веток репозитория.
--------------------------------------------

* После внесения изменений в новую ветку (черновик, тест) мы можем перенести все данные и изменения в основную или любую другую ветку. Для этого предварительно сохранив и создав новый коммит переходим в ту ветку куда хотим слить (соединить) текущие изменения с помощью команды:

```
git checkout <имя_ветки_куда_хотим_произвести_слияние> 
```

 * После перехода убеждаемся что мы на нужной нам ветке и что нет ошибок с помощью комманды `git status`.
Если все в порядке можно произвести слияние веток с помощью команды:
```
git merge <имя_ветки_для_слияния>
```
>_**Примечание:** В том случает если при слиянии в основной ветке в тех строках где были произведенны изменения был другой текст или другие изменения то может произойти конфликт как показанно на рисунке. Git предложит варианты решения (сохранить изменения, оставить текущее состояние или оставить без изменений ). В моем случае появились соответсвующие метки для ручной корректировки (т.к более старая версия операционной системы)._

![Конфликт](conflikt.png) 

+ После того как мы исправили или приняли изменения, нужно добавить изменения в файле в репозиторий и сделать новый коммит

```
git add <имя файла>

git commit -m "<новое сообщение>"
```


------------------------------------------
## 16. Удаление веткок в репозитории.
------------------------------------------
* После слияния нужной нам ветки. Не актуальную ветку можно удалить с помощью следующей команды:

``` 
git branch -d <имя_удаляемой_ветки>
```
* В случае если слияние не было произведено то Git не даст удалить такую ветку и ошибка будет выглядет так:

![ошибка слияния](error.png)

+ После того как мы произвели слияния и удаления веток и исправив ошибку обязательно делаем добавление в репозиторий и делаем новый комит для фиксации текущих изменений:

```


git add <имя файла>

git commit -m "<новое сообщение>"
```

## 17. Регистрация на GitHub.com и создание удаленного репозитория.
------------------------------------------------
* Переходим на [GitHub.com](https://github.com/) в правом верхнем углу жмем Регистрация (Registration) вводим свои данные логин и email. После этого поддтверждаем регистрацию через email.

* После регистрации мы можем создать удаленный, репозиторий для этого жмем (**+**) в правом верхнем углу и из появившихся вариантов выбираем:

**New repository**

-далее в появившемся окне в строке  

**Repository name** пишем имя репозитория

-ниже выбираем тип доступа репозитория публичный или приватный в нашем случае выбираем публичный:

**Public**

+ Все остальное в нашем случае оставляем без изменения.
Жмем:

**Create repository**

------------------------------------------------
## 18. Связываем локальный репозиторий с удаленным репозиторием (GitHub.com)
------------------------------------------------
+ Для этого создаем локальный репозиторий **(4-6 пункт)** или открываем уже готовый **File --> Open Folder** открываем папку с ранее созданным репозиторием.

+ Далее открываем на GitHub.com созданный нами удаленный репозиторий. При первом открытие GitHub.com нам предложит создать новый, внести существующий или импортировать код из другого репозитория. В нашем случии выбираем внести существующий **…or push an existing repository from the command line**.

для этого в термининале VS Code вводим поочередно предложенные нам команды:

```
git remote add origin <url-ссылка на удаленный репозиторий> - связывает локальный репозиторий с удаленным. 
```
- При первом связывании репозиториев GitHub.com запросит ваши данные Имя и Пароль для избежания не санкционированного доступа к Вашему репозиторию.

Для проверки произошло ли связывание можно использовать копанду `git remote`.

А так же с указанием путей `git remote -v`.

```
git branch -M main - переиминнование ветки master на main (если по умолчанию у Вас главная ветка main то можно эу команду пропустить).
```
```
git push -u origin main - отправка изменений из локального репозитория на удаленный. 
```
+ Обновляем страницу на GitHub.com и если все сделанно правильно, то мы увидем в удаленном репозитории файлы и папки перенесеного в него локального репозитория. 

------------------------------------------------
## 19. Внесение изменений в удаленном репозитори и перенос их на локальный.
------------------------------------------------
+ Открываем удаленный репозиторий на GitHub.com открываем нужный нам фаил и жмем на карандаш для внесения изменений в текущий фаил.
После внесения изменений в самом низу пишем комментарий к коммиту и жмем **Commit changes**

* После этого мы заходим в наш локальный репозиторий, для того что бы увидеть изменения мы можем воспользоваться командой git status

Для переноса всех изменений в локальный репозиторий воспользуемся командой:

```
git pull
```
